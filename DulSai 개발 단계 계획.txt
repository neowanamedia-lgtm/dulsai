DulSai 개발 단계 계획

[0단계] 방향 확정
- 서비스명: DulSai
- 핵심 컨셉: 사진보다 문장으로 먼저 만나는 관계 서비스
- 기본 로고: 하트 + DulSai
- 프리미엄 심볼: DS 모노그램
- 기본 프로필 아이콘: 남녀 한 쌍 아이콘 확정본 사용
- 핵심 원칙:
  1. 처음부터 얼굴 공개하지 않는다.
  2. 처음부터 나이/거주지/연락처를 공개하지 않는다.
  3. 관계 정보는 대화 단계에 따라 천천히 열린다.
  4. AI 생성 대화가 아니라 룰 기반 관계 흐름 엔진을 만든다.


[1단계] MVP 핵심 UX 설계
- 첫 화면
- 회원가입 화면
- 본인 인증 화면
- 기본 프로필 설정 화면
- 문장형 카테고리 선택 화면
- 글 작성 화면
- 글 피드 화면
- 글 상세 화면
- 답글/대화 화면
- 신고/차단 화면

우선 구현 범위:
- 좋아요 수 없음
- 팔로워 수 없음
- 조회수 없음
- 외모 중심 추천 없음
- 사진 즉시 공개 없음
- DM보다는 글/답글 중심 구조 우선


[2단계] 회원가입 구조
필수 입력:
- 휴대폰 본인 인증
- 닉네임
- 생년월일
- 성별
- 프로필 사진 업로드
- 최소 관심/감정 태그 선택

주의:
- 프로필 사진은 서버에 저장하지만 초기에는 상대에게 보여주지 않는다.
- 생년월일은 내부 연령대 계산용으로 사용한다.
- 화면에는 정확한 나이를 바로 보여주지 않는다.
- 13세 이상 허용 가능성을 기준으로 청소년 보호 구조를 함께 설계한다.


[3단계] 기본 프로필 공개 정책
초기 공개:
- 닉네임
- 성별 아이콘
- 작성한 글
- 감정/관심 태그 일부

초기 비공개:
- 얼굴 사진
- 정확한 나이
- 연령대 색상
- 거주지
- 연락처
- 학교/직장
- SNS 계정

대화 시작 후 공개 가능:
- 연령대 색상
- 연령대 표시

상호 동의 후 공개 가능:
- 프로필 사진
- 추가 자기소개 정보


[4단계] 서버 기본 구조
필요 테이블:
- users
- user_profiles
- profile_photos
- posts
- post_comments
- conversations
- conversation_messages
- relationship_states
- reports
- blocks
- moderation_logs

서버 역할:
- 사용자 계정 관리
- 휴대폰 인증 결과 저장
- 프로필 사진 저장
- 글/댓글/대화 저장
- 사용자 간 관계 단계 저장
- 신고/차단 기록 저장
- 개인정보 노출 시도 기록
- 공개 가능 단계 계산


[5단계] 문장형 카테고리 구조
카테고리 예시:
- 하루의 생각
- 외로운 순간
- 좋아하는 감정
- 인간관계에 대한 태도
- 자기만의 습관
- 조용한 고백 같은 글
- 오늘의 한 문장
- 누군가에게 하지 못한 말
- 요즘 자주 드는 마음
- 내가 편안함을 느끼는 사람
- 사랑에서 중요하게 생각하는 것

원칙:
- 카테고리는 게시판처럼 보이되, 너무 딱딱하지 않게 만든다.
- 사용자가 글을 쓰는 순간부터 자기 분위기가 드러나게 한다.
- 취미 기반보다 감정/문장 기반으로 설계한다.


[6단계] 관계 단계 엔진
Stage 0: 첫 노출
- 글과 성별 아이콘만 중심
- 얼굴/나이/지역/연락처 비공개

Stage 1: 답글 시작
- 서로 답글 가능
- 개인정보 공유 제한 유지

Stage 2: 대화 지속
- 일정 대화량 또는 일정 기간 충족
- 연령대 색상 공개 가능 안내

Stage 3: 상호 관심 확인
- 서로 관심 표시
- 프로필 사진 공개 가능 안내

Stage 4: 상호 동의
- 둘 다 동의하면 프로필 사진 공개
- 추가 정보 일부 공개 가능

Stage 5: 신뢰 단계
- 앱 내부 대화 유지 권장
- 외부 연락처 공개 여부는 매우 신중하게 설계


[7단계] 안전/차단 엔진
초기 차단 대상:
- 전화번호
- 카카오톡 ID
- 인스타그램 ID
- 텔레그램 ID
- 이메일
- 정확한 주소
- 학교명
- 직장명
- 구체적 거주지
- 직접적인 나이 공개
- 성희롱
- 욕설
- 위협
- 반복적인 만남 강요

처리 방식:
- 메시지 전송 전 검사
- 위험 문구 감지 시 전송 차단
- 사용자에게 부드러운 안내 문구 표시
- 반복 위반 시 제한
- 심각한 경우 운영 검토 대상으로 기록


[8단계] 안내 문구 엔진
목표:
- AI처럼 보이지만 실제로는 룰 기반 문구 엔진으로 구현

예시 문구:
- "조금 더 서로를 알아가 본 뒤 공개할 수 있어요."
- "DulSai에서는 문장으로 먼저 서로를 만나도록 도와드리고 있어요."
- "지금은 연락처보다 대화를 조금 더 이어가 보는 단계예요."
- "이제 서로의 연령대를 공개할 수 있어요."
- "두 분이 모두 동의하면 프로필 사진을 열 수 있어요."

원칙:
- 금지/차단 느낌을 줄이지 않는다.
- 사용자가 통제당한다고 느끼지 않게 한다.
- 관계 속도를 조율하는 안내자처럼 보이게 한다.


[9단계] MVP 개발 우선순위
1순위:
- 회원가입
- 본인 인증
- 프로필 생성
- 프로필 사진 비공개 저장
- 글 작성
- 글 피드
- 답글 기능
- 신고/차단

2순위:
- 관계 단계 엔진
- 개인정보 차단 엔진
- 연령대 공개 단계
- 프로필 사진 상호 공개 단계

3순위:
- 추천 알고리즘
- 감정 태그 기반 피드 정렬
- 대화 지속성 분석
- 프리미엄 DS 아이콘/테마
- 유료 기능 검토


[10단계] iOS 출시 전 필수 준비
- 개인정보 처리방침
- 이용약관
- 커뮤니티 가이드라인
- 청소년 보호 정책
- 신고/차단 기능
- 계정 삭제 기능
- 콘텐츠 신고 처리 정책
- 앱스토어 심사용 설명
- 13세 이상 허용 여부 최종 결정
- 프로필 사진 비공개 정책 명확화
- 연락처/개인정보 차단 정책 명확화


[11단계] 첫 출시 목표
첫 출시에서 반드시 보여줘야 할 가치:
- 사진보다 문장으로 먼저 사람을 만난다.
- 얼굴은 서로 동의해야 열린다.
- 관계 정보는 천천히 열린다.
- 연락처와 개인정보는 초기에 보호된다.
- AI가 아니라 관계 흐름 엔진이 공간의 분위기를 지킨다.

한 줄 정의:
DulSai는 외모보다 문장으로 먼저 사람을 느끼고, 관계가 천천히 열리도록 설계된 감정 기반 연결 앱이다.
